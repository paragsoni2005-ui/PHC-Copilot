# 🏥 PHC Copilot — AI-Powered Operations Assistant for Primary Health Centres

> **Google Build with AI – Code for Communities | Smart Health Track**

PHC Copilot transforms Primary Health Centre operations from passive record-keeping into **AI-assisted operational decision support**. Powered by **Google Gemini 2.5 Flash**, it analyzes real-time medicine inventory, patient footfall, and doctor attendance data to generate actionable daily briefings and smart recommendations — helping healthcare workers make better decisions in under 30 seconds.

## 🎯 Problem Statement

Primary Health Centres in India face critical operational challenges:

- **Medicine stockouts** due to manual inventory tracking
- **Unexpected patient surges** without predictive warnings
- **Doctor absenteeism** with no operational impact analysis
- **Manual report analysis** consuming valuable clinical time
- **Delayed decision-making** leading to poor patient outcomes

Current systems only record data. **PHC Copilot tells healthcare workers what to do next.**

## ✨ Key Features

### 🤖 AI Daily Operations Dashboard
A centralized command center showing real-time stats — today's patient count, critical medicine alerts, doctor availability, and an AI-generated action checklist — all at a glance.

### 💊 Smart Medicine Inventory
AI-powered stock predictions calculate remaining days of supply, shortage risk levels, and recommended reorder dates. Gemini analyzes consumption patterns and flags critical items automatically.

### 📊 Patient Footfall Analytics
Historical trend charts (7-day line, department bar, hourly area charts) with AI-powered surge prediction — tomorrow's expected patient count, peak hours, operational risk level, and staffing recommendations.

### 👥 Doctor Attendance & Impact Monitor
Real-time roster tracking with shift management (Morning/Evening/Night). AI calculates the operational impact when doctors are absent and recommends coverage adjustments.

### 📋 AI Daily Briefing & Action Checklist
One-click natural language operational summary generated by Gemini 2.5 Flash, with a dynamic action checklist that adapts based on current inventory levels, patient trends, and staffing gaps.

### 🏷️ OPD Patient Registration
Complete patient intake system with department routing, symptom capture, visit type tracking, and a searchable/filterable patient registry with real-time Firestore sync.

### 🛡️ Resilient Offline Mode
When the network or Gemini API is unavailable, a **local rule-based decision engine** automatically takes over — generating checklist tasks and operational alerts using deterministic healthcare business rules, ensuring the app never goes dark.

## 🚀 Getting Started

### Prerequisites

- **Node.js** 18+ installed
- A **Firebase** project with Firestore enabled
- A **Google Gemini API key** from [Google AI Studio](https://aistudio.google.com/)

### 1. Clone the repository

```bash
git clone https://github.com/YOUR_USERNAME/phc-copilot.git
cd phc-copilot
```

### 2. Install dependencies

```bash
npm install
```

### 3. Configure environment variables

Copy the example file and fill in your credentials:

```bash
cp .env.example .env.local
```

Edit `.env.local`:

```env
GEMINI_API_KEY=your_gemini_api_key

NEXT_PUBLIC_FIREBASE_API_KEY=your_firebase_api_key
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=your_project.firebaseapp.com
NEXT_PUBLIC_FIREBASE_PROJECT_ID=your_project_id
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=your_project.firebasestorage.app
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=your_sender_id
NEXT_PUBLIC_FIREBASE_APP_ID=your_app_id
```

### 4. Configure Firestore Security Rules

In your Firebase Console → Firestore → Rules, use:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId}/{document=**} {
      allow read, write: if true;
    }
    match /seedVerification/{doc} {
      allow read, write: if true;
    }
  }
}
```

### 5. Run locally

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000)

> **Note:** The app automatically seeds representative PHC data into Firestore on first visit — no manual database setup required.

## 🧠 How AI Integration Works

### Gemini 2.5 Flash Integration

PHC Copilot uses Google Gemini in three critical workflows:

1. **Daily Briefing Generation** — Synthesizes medicine stock levels, patient trends, and roster data into a natural language operational summary with prioritized action items.

2. **Dynamic Checklist Generation** — Queries real-time Firestore metrics and generates context-aware action items with structured JSON output schemas (category, priority, reason, estimated impact).

3. **Medicine Predictions** — Calculates remaining stock days, shortage risk, and optimal reorder dates based on consumption patterns.

### Resilient Fallback Architecture

If Gemini API is unavailable (network failure, quota exhaustion, or API errors):

- The system automatically switches to a **local rule-based decision engine**
- Healthcare business rules generate deterministic alerts:
  - Symptom outbreak detection (≥15% surge over 7-day rolling average)
  - Low stock warnings (medicines below 50-unit threshold)
  - Roster gap alerts (absent doctors during active shifts)
- The UI displays a **"✓ Local Decision Engine"** badge so users know the source
- An **offline detection banner** appears: *"Offline Mode — Using Local Rule Engine"*

## 🔒 Security

- Gemini API key is stored **server-side only** (`process.env.GEMINI_API_KEY`) — never exposed to the browser
- Firebase credentials use `NEXT_PUBLIC_` prefix (client-safe, restricted by Firebase Security Rules)
- Vercel deployment includes security headers (X-Content-Type-Options, X-Frame-Options, X-XSS-Protection)
- User data is sandboxed per authenticated user ID in Firestore

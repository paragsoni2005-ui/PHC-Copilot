# Phase 6.1 Context Map: OPD Registration Form Polish & Auth Disable

## Goal
The goal of this phase is to disable the authentication gate for seamless testing, remove mock elements from the landing page, and polish the OPD patient registration form into a rapid-entry "Quick Registration Form" with selectable symptom chips.

## Key Changes
1. **Disable Authentication**:
   - Update `AuthContext.tsx` to bypass Firebase authentication listeners, exposing a default mock user and role by default.
   - Remove route redirection rules from `AppShell.tsx` and `AuthContext.tsx`.
   - Update `AppShell.tsx` to show a unified sidebar layout containing both the Medical Officer options and the OPD registration view.
2. **Landing Page Refactor**:
   - Remove the "No real-time visibility" card.
   - Remove the mock OPD Patient Registration preview card.
   - Route CTA buttons directly to `/opd-registration` and `/dashboard` respectively.
3. **Quick Registration Form**:
   - Modify `/opd-registration` to use the updated Quick Registration Form.
   - Adjust Department dropdown selections to: `General OPD`, `Pediatrics`, `ANC`, and `Immunization`.
   - Replace the Symptoms free-text box with interactive, toggleable symptom chips: `Fever`, `Cough`, `Cold`, `Headache`, `Body Pain`, `Stomach Pain`, `Injury`, and `Other`.
   - Support a text input field for write-in symptoms when `Other` is selected.
   - Concatenate active symptom selections into a comma-separated string upon submission and sync to Firestore.
